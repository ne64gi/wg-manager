# Release Notes: `v1.1.4`

`wg-studio v1.1.4` continues the `1.1` line with a mix of frontend cleanup and runtime/ops follow-up work.

## Highlights

- split the frontend API layer into focused modules under `frontend/src/lib/api/`
- refined the login screen controls, theming, and language/theme carry-over behavior
- moved more artifact/config write mechanics behind the shared runtime service facade
- expanded wrapper entrypoints with `health` and `smoke` commands

## Runtime/Ops Follow-Up

`v1.1.4` still keeps the same core `1.1` goal: reduce coupling without jumping into broad cross-platform abstraction too early.

Changes in this slice:

- `config_generation` now writes server config, peer config, and peer QR files through `RuntimeService`
- `RuntimeService` now owns a small artifact-facing public surface in addition to runtime read/apply behavior
- wrapper scripts can now run:
  - `health`
  - `smoke`

The only supported runtime adapter is still `docker_container`.

## Frontend Cleanup

- the previous monolithic `frontend/src/lib/api.ts` file is now split into smaller modules by domain
- the login screen now has:
  - icon-only display settings button
  - easier-to-hit theme toggle
  - flag-labelled language selector
  - username/password icons
  - password visibility toggle
  - calmer dark-mode input styling

## Validation

- backend pytest: passed
- Playwright smoke: passed
- `docker compose build wg-studio-web`: passed
