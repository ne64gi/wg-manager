# Release Notes: `v1.1.3`

`wg-studio v1.1.3` continues the `1.1` runtime-separation line while tightening operator entry points and smoothing a few desktop GUI edges.

## Highlights

- introduced a shared runtime service facade for `apply` and `status`
- reduced direct service-level dependency on the runtime adapter constructor path
- added logical targets to `scripts/stack.sh` and `scripts/stack.ps1`
- improved dashboard desktop layout and desktop sidebar collapse behavior
- added favicon support for the web UI

## Runtime Separation Progress

`v1.1.3` still does not aim for cross-platform runtime support yet.

Instead, it keeps pushing toward separability:

- runtime reads and apply operations now go through a shared runtime-facing service
- status handling is more resilient when the runtime is temporarily unavailable
- local startup is less brittle when older PostgreSQL volumes are reused

The currently supported runtime adapter remains `docker_container`.

## Operator Workflow Changes

Wrapper scripts now understand logical targets for `up`, `build`, and `restart`:

```bash
./scripts/stack.sh up core
./scripts/stack.sh build api
./scripts/stack.sh restart web
```

On PowerShell:

```powershell
pwsh ./scripts/stack.ps1 up core
pwsh ./scripts/stack.ps1 build api
pwsh ./scripts/stack.ps1 restart web
```

This is still a preparation step, not a full deployment redesign.

## UI Improvements

- the dashboard uses desktop space more efficiently around the traffic/online summary area
- the desktop sidebar can collapse into icon-only navigation
- the settings checkbox row spacing is cleaner
- browsers no longer log a missing favicon error for the main app shell
