# Release Notes: `v1.3.0`

`wg-studio v1.3.0` is a UX and visibility release focused on helping operators understand current WireGuard state faster.

## Highlights

- replaced the dashboard side card with a `runtime / system` panel that shows runtime status, uptime, container identity, and sync-related timestamps
- reorganized the main dashboard panels behind slot-based entries so future replacement and per-user layout work has a stable seam
- upgraded the traffic timeline with range switching, axes, legend, and expanded view support
- added a direct topology API for `Group -> User -> Peer`, powering the new `/network` graph and preparing the data contract for future topology views
- added traffic snapshot retention settings and automatic cleanup of expired history rows
- added a dedicated `/network` page that renders `Server -> Group box -> User -> Peer` as an operator-friendly relationship graph
- isolated Playwright smoke runs from the operator stack by moving them to a dedicated compose project and volumes
- moved the sidebar user popover to a React Portal so it no longer gets clipped by sidebar or content overflow

## Why It Matters

This release is about making the control plane easier to read during normal operation, not only when drift or failure is present.

Operators can now see runtime uptime and health context even in a clean state, while the topology and timeline views provide a clearer mental model of the current network.

This release also hardens the day-to-day developer and operator experience by isolating E2E smoke runs from production state and by replacing fragile in-layout popovers with a portal-based implementation.

## Operator Notes

- a new `Traffic history retention (days)` setting is available under `Settings > Language and time`
- expired `peer_traffic_snapshots` rows are deleted automatically during snapshot capture
- the topology API is now used by both the lightweight dashboard preview and the `/network` relationship graph
- Playwright smoke now expects the isolated `wg-studio-e2e` compose project instead of touching the operator stack directly
