# Release Notes: `v1.2.2`

`wg-studio v1.2.2` is a stability and hardening release focused on runtime config drift and platform compatibility.

## Highlights

- fixed runtime drift after interface recreation by reconciling WireGuard peer `AllowedIPs` and default route state during apply
- added automatic server config re-apply when initial setting `server_address` or `server_dns` changes
- added persistent `server_address` and `server_dns` in Initial Settings API/UI
- removed hard-coded Docker API version path; now uses `docker_api_version` setting for compatibility with Docker versions

## Why It Matters

This release addresses urgent production failure modes where `wg0` could be recreated without the desired route/peer config, and where incorrect Docker API version path breaks controller startup in variant Docker setups.

By exposing server address + DNS in settings and applying config automatically, operators get safer control-plane state convergence.

## Operator Notes

- make sure to update front-end Settings page to set the configured server endpoint and DNS if required
- restart is no longer required for Docker API version changes; setting is applied at runtime
- this release is intended as a resilient fix layer on top of the previous `1.2.1` frontend polish work
