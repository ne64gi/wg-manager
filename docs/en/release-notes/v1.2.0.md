# Release Notes: `v1.2.0`

`wg-studio v1.2.0` starts the frontend modernization-prep phase.

This is not a framework migration. It is the first release that intentionally creates seams so the current Vite/React app can later move more safely toward a Next-ready structure.

## Highlights

- extracted route composition out of [`App.tsx`](../../frontend/src/App.tsx) into [`AppRoutes.tsx`](../../frontend/src/app/AppRoutes.tsx)
- moved authenticated shell wiring into [`AppShell.tsx`](../../frontend/src/app/AppShell.tsx) so reusable layout code no longer needs to know about router primitives directly
- moved preview theme and preview locale behavior into dedicated frontend preference modules
- introduced a small browser storage helper so `localStorage` access starts leaving feature code
- converted document theme/lang synchronization into a dedicated hook
- started grouping browser-only confirm/download/copy actions behind shared helpers instead of page-local DOM code
- moved dashboard data loading and apply-side effects behind a dedicated dashboard module hook
- moved settings page query/mutation state behind a dedicated GUI module hook
- moved logs-page filter/paging/query orchestration behind a dedicated GUI module hook
- moved login-page submit/setup/preview state behind a dedicated auth module hook
- moved group, user, and peer page orchestration behind feature-level hooks so those pages are closer to composition shells
- added lightweight search controls to group and user management lists so larger datasets are easier to scan without leaving the page
- added a lightweight dashboard topology preview for `Group -> User -> Peer` relationships as a structural placeholder ahead of later graph work
- added smoke coverage that checks login-screen display settings carry into the authenticated app
- added a lightweight `Expand all / Collapse all` control for grouped dashboard traffic cards

## Why It Matters

`v1.2.0` is the first frontend-focused separation release.

The goal is not visible redesign. The goal is to reduce coupling before later work on:

- Next-ready layout structure
- browser-only API isolation
- router-specific dependency cleanup

## Operator Notes

- no operator workflow was intentionally changed in this release
- login display settings should still carry into the authenticated UI, and this flow is now smoke-tested
- grouped dashboard traffic now supports quick expand/collapse without changing the underlying workflow
- the visible UI should remain effectively the same while page/controller responsibilities are now thinner behind the scenes
