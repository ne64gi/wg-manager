# Release Notes v1.1.5

## Summary

`v1.1.5` makes the operator entrypoints safer to use during startup and adds a small runtime-facing visibility improvement in the GUI.

## Changes

- `scripts/stack.sh` and `scripts/stack.ps1` now expose a `wait` command for bounded API readiness checks
- `health` now verifies API readiness first and then checks web reachability
- `smoke` now reuses the readiness preflight before running Playwright
- the Playwright smoke suite waits for a stable UI root more deterministically and uses canonical group network values
- `Build information` in Settings now shows the active runtime adapter

## Why It Matters

- fewer startup surprises when operators run wrapper commands immediately after `up`
- clearer distinction between “API is not ready yet” and “the smoke suite itself failed”
- better visibility into which runtime adapter the current stack is using
