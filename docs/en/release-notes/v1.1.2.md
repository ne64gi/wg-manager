# Release Notes: `v1.1.2`

`wg-studio v1.1.2` continues the `1.1` runtime-separation line while still adding a small operator-facing improvement in the GUI.

## Highlights

- pushed more WireGuard runtime mechanics into [`app/runtime/`](../../app/runtime/)
- split `wg show ... dump` parsing into [`app/runtime/dump.py`](../../app/runtime/dump.py)
- added a shared runtime read facade in [`app/runtime/service.py`](../../app/runtime/service.py)
- expressed artifact storage as a protocol with a local filesystem implementation
- removed the old `app/services/docker_runtime.py` compatibility shim
- replaced the separate dashboard user-traffic panel with inline user traffic under each group row
- added desktop sidebar collapse for the GUI
- added a bundled favicon so browsers stop requesting a missing icon asset
- extended Playwright smoke coverage for the dashboard traffic accordion

## Why This Release Matters

`v1.1.2` is still primarily an internal-structure release.

The important change is not cross-platform support yet. The important change is that more Linux/Docker-shaped behavior now lives behind clearer runtime boundaries:

- runtime dump parsing is no longer mixed into status orchestration
- artifact storage is easier to swap later
- the old Docker-specific service shim is no longer a required public path

This keeps `1.1` moving toward separability without trying to finish portability too early.

## Operator Notes

- the only supported runtime adapter is still `docker_container`
- stack startup remains unchanged:

```bash
docker compose up -d --build
```

- launch wrappers remain available as early-prep helpers:

```bash
./scripts/stack.sh up
pwsh ./scripts/stack.ps1 up
```

## Validation

- runtime-focused backend tests pass
- frontend build passes
- Playwright smoke now also verifies dashboard group traffic expansion
