# リリースノート: `v1.1.3`

`wg-studio v1.1.3` は、`1.1` 系の runtime separation をさらに進めつつ、運用入口とデスクトップ GUI の細かな使い勝手を整えた版です。

## 主な変更

- `apply` と `status` に共通の runtime service facade を導入
- service 層から runtime adapter の直接解決をさらに減らした
- `scripts/stack.sh` / `scripts/stack.ps1` に論理ターゲットを追加
- Dashboard のデスクトップ配置とサイドバー折りたたみを改善
- Web UI に favicon を追加

## Runtime Separation の進み方

`v1.1.3` でも、まだクロスプラットフォーム対応そのものは目標ではありません。

今回の主眼は引き続き separability です。

- runtime 読み取りと apply が共通の runtime service を通るようになった
- runtime が一時的に不安定でも status 系が落ちにくくなった
- 古い PostgreSQL volume を再利用する開発環境でも起動が壊れにくくなった

現時点で対応している runtime adapter は引き続き `docker_container` のみです。

## 運用入口の変更

wrapper script で `up` / `build` / `restart` に論理ターゲットを指定できるようになりました。

```bash
./scripts/stack.sh up core
./scripts/stack.sh build api
./scripts/stack.sh restart web
```

PowerShell では:

```powershell
pwsh ./scripts/stack.ps1 up core
pwsh ./scripts/stack.ps1 build api
pwsh ./scripts/stack.ps1 restart web
```

これはあくまで入口整理の一歩であり、大きなデプロイ再設計ではありません。

## UI 改善

- Dashboard の右側領域を使ってデスクトップ表示を整理
- デスクトップのサイドバーをアイコン中心に折りたためるようにした
- 設定画面のチェックボックス行の見た目を調整
- favicon 未設定のブラウザエラーを解消
