# リリースノート: `v1.2.0`

`wg-studio v1.2.0` は、frontend modernization prep の最初の版です。

これは Next.js への移行版ではありません。今の Vite/React アプリを、将来より安全に Next-ready な構造へ寄せるための最初の切れ目を入れる版です。

## 主な内容

- [`App.tsx`](../../frontend/src/App.tsx) に直接置かれていた route 合成を [`AppRoutes.tsx`](../../frontend/src/app/AppRoutes.tsx) へ分離
- 認証後の shell 合成を [`AppShell.tsx`](../../frontend/src/app/AppShell.tsx) へ寄せ、再利用 layout から router 直結を減らした
- preview theme / preview locale の処理を専用 module へ移動
- `localStorage` 利用を少しずつ feature code から離すための browser storage helper を追加
- document の theme/lang 同期を専用 hook に分離
- `confirm / download / copy` のような browser-only 処理を page ローカル実装から shared helper へ寄せ始めた
- Dashboard の query / mutation と apply 副作用を専用 module hook へ切り出した
- Settings 画面の query / mutation / form state を専用 GUI module hook へ切り出した
- Logs 画面の filter / paging / query を専用 GUI module hook へ切り出した
- ログイン画面の submit / setup / preview state を専用 auth module hook へ切り出した
- Groups / Users / Peers の query / mutation / form orchestration を feature module hook へ寄せ、page を composition 寄りにした
- グループ管理とユーザー管理に軽い検索導線を追加し、ページを離れず一覧を絞り込みやすくした
- Dashboard に `Group -> User` 構造を軽く確認できる構成プレビューを追加し、Peer は件数で示す形にした
- ログイン画面で選んだ表示設定が、認証後 UI と Settings まで引き継がれることを smoke で確認するようにした
- 保存した theme / locale が login preview、ログイン後の UI、Settings 画面でずれないように修正した
- Settings から interface-level MTU を入力できるようにし、設定時は生成される WireGuard config に反映するようにした
- Dashboard は構成プレビューを残し、重複しやすかったグループ別トラフィック枠は外した

## この版の意味

`v1.2.0` は、見た目を大きく変える版ではありません。

主目的は、今後の

- Next-ready な layout 整理
- browser-only API の隔離
- router 依存の整理

をやりやすくするために、先に frontend の責務を切り始めることです。

## 運用メモ

- この版では見た目以上に frontend の責務分離が進みました
- ログイン画面の表示設定は、preview とログイン後 UI と Settings 保存状態がそろうようになりました
- Settings では任意の interface MTU を持てるようになり、生成される WireGuard config に埋め込まれます
- Dashboard は drift/apply 状態と構成プレビューを優先する構成に寄せています
