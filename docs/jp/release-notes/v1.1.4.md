# リリースノート: `v1.1.4`

`wg-studio v1.1.4` は、`1.1` 系の流れを保ったまま、フロントの整理と runtime/ops の追い込みを組み合わせた版です。

## 主な変更

- frontend の API 層を `frontend/src/lib/api/` 配下の小さなモジュールへ分割
- ログイン画面の操作性、見た目、テーマ/言語の引き継ぎを改善
- artifact/config 書き込み責務を shared runtime service facade の内側へさらに寄せた
- wrapper script に `health` と `smoke` を追加

## Runtime/Ops の進み方

`v1.1.4` でも引き続き、広い cross-platform 抽象化に飛ぶのではなく、coupling を少しずつ剥がすことを優先しています。

今回の変更:

- `config_generation` は server config、peer config、peer QR を `RuntimeService` 経由で書き出すようになった
- `RuntimeService` は runtime の read/apply だけでなく、artifact 書き込みの小さな公開面も持つようになった
- wrapper script で次を直接実行できるようになった
  - `health`
  - `smoke`

現時点で対応している runtime adapter は引き続き `docker_container` のみです。

## フロント整理

- 以前の大きな `frontend/src/lib/api.ts` は、ドメイン別の小さなモジュールへ分割された
- ログイン画面では次を改善した
  - 表示設定ボタンのアイコン化
  - 押しやすい theme toggle
  - 国旗付き言語選択
  - username/password アイコン
  - パスワード表示切り替え
  - dark mode 入力欄の色調整

## 検証

- backend pytest: 成功
- Playwright smoke: 成功
- `docker compose build wg-studio-web`: 成功
