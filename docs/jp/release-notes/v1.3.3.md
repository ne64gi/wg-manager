# リリースノート: `v1.3.3`

`wg-studio v1.3.3` は、`1.3.0` で始めた UX / visibility 改善を、実運用で使いやすい状態まで詰めた仕上げの版です。

## 主な変更

- `Apply` は即実行ではなく、生成されるサーバー設定の差分プレビューを確認してから確定できるよう改善
- `DB full backup / restore` と `state export / import` の CLI / host script を追加し、復旧と移送の経路を明確化
- `pytest-safe.sh` と非テスト DB ガードを追加し、本番 DB に test や reset が向く事故を防止
- `/network` を相関グラフとして強化し、`Group / User / Peer` の選択、focus、traffic hotspot、`Rx / Tx`、endpoint、filter 保持を追加
- `/network` 上で選択中ノードの `有効 / 無効` を切り替えられるようにし、運用導線を強化
- network graph の視覚ルールを整理し、`選択中 / 非選択 / 無効 / 未表示 / オンライン` の違いが分かる見え方へ調整
- 検索文言と実際の検索対象を揃え、Group / User / Peer 一覧のヘッダで昇順 / 降順を切り替え可能に変更
- `Peer reissue`、state import などの危険操作に確認文や警告を追加
- 共通の shell / panel / card / tab / nav の質感を整理し、より一般的な管理 UI に寄せた見た目へ調整

## ねらい

この版の目的は、見た目を足すことではなく、`wg-studio` を日常運用で安心して触れる状態に寄せることです。

とくに今回は、

- 変更前に差分を見て判断できること
- 誤って本番 DB を壊しにくいこと
- ネットワーク構成を「読める」だけでなく、その場で判断と操作に使えること

を重視しています。

## 運用メモ

- 破壊的な DB 操作や import / restore / manual SQL の前は、`scripts/backup-db.sh` による full backup が前提です
- `state export / import` は論理移送用であり、recovery-grade backup の代替ではありません
- `./scripts/stack.sh pytest ...` は安全な `*_test` / `*_e2e` DB に向けて実行されます
- `/network` の filter 状態は localStorage に保持され、次回表示時に復元されます
