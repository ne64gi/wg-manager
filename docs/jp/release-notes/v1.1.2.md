# リリースノート: `v1.1.2`

`wg-studio v1.1.2` は、`1.1` 系の runtime separation をもう一段進めつつ、GUI には小さくても実用的な改善を入れた版です。

## 主な内容

- WireGuard runtime 関連の責務をさらに [`app/runtime/`](../../app/runtime/) へ寄せた
- `wg show ... dump` の解釈を [`app/runtime/dump.py`](../../app/runtime/dump.py) に分離
- runtime 読み取りの共通入口として [`app/runtime/service.py`](../../app/runtime/service.py) を追加
- artifact storage を protocol + local filesystem 実装の形に整理
- 旧 `app/services/docker_runtime.py` 互換 shim を削除
- Dashboard の `User traffic` 独立パネルをやめ、各 group の中に user traffic を展開表示する形へ変更
- GUI にデスクトップ用サイドバー折りたたみを追加
- favicon を追加して、ブラウザの missing icon エラーを止めた
- Dashboard の group traffic 展開を Playwright smoke で確認するようにした

## この版の意味

`v1.1.2` の主題は、やはり portability 完成ではありません。

主眼は、Linux / Docker 依存の runtime 形状を core の外へ押し出し続けることです。

- runtime dump 解釈を status orchestration から剥がす
- artifact storage の差し替え余地を増やす
- Docker 依存の古い service shim を不要にする

ことで、`1.1` の separability を前へ進めています。

## 運用メモ

- 対応している runtime adapter は引き続き `docker_container` のみです
- 通常の起動コマンドは今まで通りです

```bash
docker compose up -d --build
```

- 起動 wrapper は引き続き将来の入口整理に向けた補助です

```bash
./scripts/stack.sh up
pwsh ./scripts/stack.ps1 up
```

## 検証

- runtime 周辺の backend テストが通過
- frontend build が通過
- Playwright smoke に Dashboard の group traffic 展開確認を追加
