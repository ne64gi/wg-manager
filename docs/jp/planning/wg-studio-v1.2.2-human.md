# wg-studio v1.2.2 作業内容（人間向け）

## このバージョンでやること
v1.2.2 では、機能追加を広げる前に、まず足元を固める。  
今回やるのは次の3つ。

1. さっき出たバグの修正
2. 汎用部分の Common 化
3. 命名規則の統一

この3つまでを v1.2.2 の作業内容とする。

---

## 1. バグ修正
今回の実運用・検証で出た不具合を先に潰す。  
ここは最優先。

### 主な対象
- `wg0` が無い / 消えた時の挙動
- interface address のズレ
- route が `ens3` に出る問題
- Docker API version 固定問題
- config と runtime がズレているのに UI 上だと正常に見える問題
- endpoint と interface address の責務が曖昧だった問題

### 目標
- 今回踏んだ障害を再発させない
- apply 後に「ちゃんと動く」確率を上げる
- 少なくとも実行環境での基本動作を安定させる

---

## 2. 汎用パーツを Common 化
wg-studio 固有ではない部分を切り出して、今後の別システムでも使える形に整理する。

### 今回 Common にしたいもの
- ログイン画面
- 画面の枠組み（レイアウト）
- サイドバー / ヘッダ / ページタイトル
- ダッシュボードの共通パーツ
- 共通テーブル
- ステータス表示
- 監査ログの基盤

### ねらい
- wg-studio を母艦にしすぎない
- 今後の別プロダクトに流用できるようにする
- UI / Auth / Audit を共通資産にする

### 今回やらないもの
- WireGuard 固有ロジック
- peer reveal
- AllowedIPs 生成
- runtime 制御
- plugin 化そのもの

---

## 3. 命名規則を統一
今かなり命名が散っているので、ここで一度整理する。

### 対象
- ファイル名
- ディレクトリ名
- コンポーネント名
- API 名
- state 名
- 設定キー
- UI 表示名との対応

### 目標
- 初見で責務が分かる名前にする
- Common と wg-studio 固有を見分けやすくする
- 今後の開発で迷わないようにする

### 方針
- 先にルールを決める
- そのルールに沿って直す
- 中途半端に混在させない

---

## 優先順位
順番は固定。

1. バグ修正
2. Common 化
3. 命名規則統一

---

## v1.2.2 の位置付け
v1.2.2 は機能追加メインの版ではなく、  
**v1.3 に入る前の土台整理版** とする。

---

## 完了イメージ
v1.2.2 完了時点で、最低限こうなっていたい。

- 今回出た不具合が潰れている
- 汎用部分が Common として切り出されている
- 命名規則が整理されている
- 次に v1.3 へ進むための土台ができている
