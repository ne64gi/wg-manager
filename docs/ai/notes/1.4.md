# wg-studio 1.4 (AI reference)

Theme: Access Control + Mandatory Audit

## Goal
Introduce enforceable RBAC and non-bypassable audit logging,
ensuring all critical actions are authorized and traceable.

---

## RBAC model

Roles:
- admin: full access
- group_admin: limited to own group

Scope:
- own: resource.group_id == actor.group_id
- other: otherwise

---

## Restrictions

group_admin MUST NOT:
- execute config.apply
- change roles
- access other groups
- modify global/system configuration

admin:
- unrestricted access
- responsible for boundary definition and runtime apply

---

## Apply policy

- config.apply is admin-only
- group_admin may modify desired state only
- runtime changes are centrally controlled

---

## Policy model

decision = POLICY[action][role][scope]

Decisions:
- ALLOW
- DENY

Scope rules:
- scope must be evaluated for all resource-bound actions
- admin bypasses scope restriction

---

## Audit model

Two independent log streams:

1. app log
- configurable via log level
- operational/debug purposes

2. audit log
- mandatory events only
- MUST NOT be disabled by configuration
- independent from app log

---

## Mandatory audit events

Authentication:
- auth.login.success
- auth.login.failure
- auth.logout

Operations (all require outcome):
- peer.create
- peer.delete
- peer.reveal
- peer.regenerate
- config.apply
- role.change
- graph.view
- drilldown.view

---

## Audit rules

- All events MUST include outcome
- outcome ∈ {ALLOW, DENY, FAILURE}
- DENY events MUST always be recorded
- sensitive read events MUST always be recorded
- correlation_id is mandatory for all events
- audit log MUST NOT be affected by log level settings

---

## Audit schema (minimal)

Required fields:
- timestamp
- event
- actor {login_user_id, role, group_id}
- resource {type, id, group_id}
- outcome
- reason
- correlation_id

Optional:
- client_ip

---

## Completion criteria

- RBAC is enforced for all actions
- config.apply restricted to admin
- policy table is implemented and used in middleware
- audit log is separated from app log
- mandatory events are always recorded
- correlation_id is attached to all events
- DENY and sensitive read events are guaranteed to be logged

---

## Non-goals

- no retention policy enforcement
- no external logging integration
- no append-only or WORM guarantee
- no MFA / OIDC / SAML (deferred to future versions)
