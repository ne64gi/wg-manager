# wg-studio 1.3 (AI reference)

Theme: Frontend modernization preparation + UI responsibility structuring

## Goal
Prepare frontend architecture for future framework migration (Next.js or equivalent)
while establishing correct UI responsibility boundaries and introducing foundational features.

## Constraints
- Do NOT introduce framework-specific features
- Maintain current routing behavior
- Avoid breaking API contracts

---

## Architecture changes

### 1. Separation of concerns
- Separate page composition from reusable UI components

### 2. Router decoupling
- Shared components must not depend on router

### 3. Browser API isolation
- Isolate browser-only APIs (window, localStorage)

### 4. Data fetching separation
- Separate data fetching from presentation layer

### 5. Layout boundaries
- Introduce layout-level structure (Next-ready)

---

## UI responsibility structuring

### 1. Auth vs Settings separation
- Authentication / access control concerns must be separated from system settings
- login user management must NOT be treated as system configuration

### 2. Settings scope definition
Settings must only include:
- system configuration
- runtime configuration
- logging configuration
- UI-related configuration

### 3. Auth scope definition
Authentication domain includes:
- login user management
- role management
- access control (future RBAC integration)

### 4. Structural direction
- login user management must be moved out of Settings
- prepare for future dedicated Access/Auth section

---

## Feature / UI changes

### 1. Network page foundation
- Introduce a dedicated "Network" page
- Designed for topology visualization (future 2D/3D graph)
- Current state: structural + data-ready

### 2. Settings UI restructuring
- Replace flat card layout with tab-based navigation
- Organize configuration into logical groups

### 3. Dashboard improvements
- Desktop layout refinement
- Sidebar icon-only collapse support
- Navigation consistency improvements

### 4. Visual improvements
- favicon added
- UI consistency improvements

---

## Expected outcome
- Frontend becomes modular and portable
- UI responsibility boundaries are clearly defined
- Future RBAC (1.4) integrates cleanly
- Future network visualization is supported
- Migration to Next.js becomes feasible without rewrite

---

## Non-goals
- No Next.js migration
- No SSR implementation
- No routing replacemen.
